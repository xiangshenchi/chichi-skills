**标题：我做了一个帮自己写毕业论文文档的 AI Skill**

写这篇文章的时候，我的毕业论文刚刚提交。

我想聊聊这几个月里一个很具体的痛苦：**代码写完了，但论文文档完全不知道从哪下手。**

项目跑得好好的，功能也都实现了。可一坐下来要写"系统架构设计"，脑子就开始空白。不是不知道自己做了什么，而是——怎么把"我写了一堆 Python 文件"翻译成"本系统采用分层架构，各模块职责清晰"这种论文语气？

我试过让 Claude 直接帮我写，但每次都要重新解释项目背景、贴代码片段、说明哪个文件是干什么的。问题是，我和它之间有大量的对话历史，里面其实已经包含了项目的所有决策和讨论——但每次新开对话，它全忘了。

所以我做了一个东西：**thesis-doc-agent**。

它是一个 Claude Skill，本质上是一套精心设计的指令集，告诉 AI agent 在面对毕业论文场景时应该怎么工作。你把代码仓库丢给它，或者把你们之间的对话历史给它看，它会自动：

- 扫描项目文件结构，理解模块分工
- 从对话历史里提取需求和设计决策
- 按论文章节生成对应文档（需求分析、架构设计、数据库设计、模块实现、测试报告……）
- **所有代码引用都强制标注文件路径和行号**，不会出现"在某个文件里"这种废话

最后这一点是我花了最多精力设计的。因为我发现，AI 在写文档时最容易犯的错误不是内容错，而是**引用模糊**——说了半天，你不知道它在描述哪个文件的哪段代码。对写论文来说，这是致命的，答辩老师是会追问的。

现在它已经开源在 GitHub 上了，如果你也正在经历相似的痛苦，可以去看看。

→ GitHub 链接

不确定这对别人有没有用，但至少对我来说，它让这件事从"不知道从哪开始"变成了"跑一遍，然后改改"。

## 版本三：CSDN 版
### 风格：技术向，有结构，偏教程但保留个人视角

---

**标题：我开源了一个 Claude Skill：让 AI 从代码库自动生成毕业论文文档**

## 背景：一个非常具体的问题

毕设阶段，我遇到了一个相信很多同学都有过的困境：**项目代码基本完成，但软件工程文档从零开始写，效率极低。**

直接让 Claude 帮写，结果是每次都要重新交代项目背景，而且生成的内容引用模糊——"在用户模块中实现了登录功能"这种描述，答辩时根本无法支撑具体追问。

我需要的是一个工具，能够：
1. 理解我的代码结构和对话历史
2. 输出带有精确文件路径引用的文档
3. 语言风格直接符合学术论文规范

于是我做了 **thesis-doc-agent**。

## 它是什么

thesis-doc-agent 是一个 Claude Skill（适用于 Claude Code 及兼容 agent 环境的指令集文件）。核心是一份 `SKILL.md`，定义了 agent 在毕业论文文档生成场景下的完整工作流程，配合 `references/` 目录下的各类文档模板使用。

**支持生成的文档类型：**

| 文档 | 对应章节 |
|------|--------|
| 需求分析 | 第二章 |
| 系统架构设计 | 第三章 |
| 数据库设计（含 ER 图） | 第三章 |
| API 接口文档 | 第三章/附录 |
| 模块详细设计 | 第四章 |
| 测试方案 | 第五章 |
| 总结与展望 | 第六章 |

## 核心设计：强制代码溯源

这是我认为最关键的设计点。

普通提示词让 AI 写文档，很容易产生这样的内容：

> ❌ "系统在认证模块中实现了 JWT 登录功能。"

thesis-doc-agent 的指令明确禁止这类模糊引用，所有代码相关描述必须遵循格式：

> ✅ **来源：** `src/api/auth/router.py`，第 45–78 行  
> JWT 登录接口，对用户凭据进行 bcrypt 验证后签发 Token，有效期 24 小时。

这个规范在 `SKILL.md` 里通过正面示例和禁止写法双重约束来执行，实测效果稳定。

## 工作流程

```
1. 信息采集
   ├── 扫描项目文件结构（自动 find 命令）
   ├── 从对话历史提取需求与设计决策
   └── 分析核心模块的接口与依赖关系

2. 按需生成
   ├── 加载对应文档模板（references/）
   ├── 填充代码内容，强制标注路径
   └── 输出 Mermaid 图表 + 学术风格正文

3. 质量自检
   ├── 学术规范检查
   ├── 代码溯源完整性检查
   └── 内容完整性检查
```

## 安装使用

```bash
git clone https://github.com/your-username/thesis-doc-agent.git
cp -r thesis-doc-agent ~/.claude/skills/
```

安装后在 Claude Code 中直接自然语言触发：

```
"帮我生成系统架构设计文档"
"分析我的数据库模型，生成 ER 图和表结构说明"
"根据我们的对话历史整理需求分析"
```
