# 把代码仓库扔给 Claude，它给我输出了一份完整的测试报告｜我做了个 Skill 开源了

最近做了个好玩的东西，分享给同样在折腾 Claude 的朋友们 👇

---

## 起因

上周接手了一个朋友的项目，几千行 Python，没有文档，测试覆盖率接近 0。

我想快速搞清楚这玩意儿的质量怎么样——有没有安全问题、依赖有没有漏洞、哪些函数写得太烂需要重构——但手动一个个工具跑太麻烦了。

然后我就想：能不能让 Claude 帮我自动完成这件事？

**结果还真做出来了。**

---

## 做了什么

我写了一个 Claude Skill，叫 `repo-test-analyst`。

用法超简单：把项目压缩包丢给 Claude，说一句"帮我分析一下"，它会自己：

**① 识别项目用了什么技术**

Python？TypeScript？还是微服务？它会通过 `package.json`、`pyproject.toml`、`go.mod` 这类文件自动判断，不用你告诉它。

**② 选合适的测试策略**

不同项目适合不同的测试方法。一个 REST API 和一个前端 SPA 该测的东西完全不一样，Skill 会根据项目类型自动决定用哪些工具、做哪些检查。

**③ 跑分析、扫漏洞**

静态分析、安全扫描、依赖审计、代码复杂度……该查的都查。

**④ 输出报告**

最终生成一套 Markdown 报告，包括：

- 📊 **综合健康度报告**：总评分 + 最重要的 Top 3 问题
- 🧹 **代码质量报告**：哪些函数太复杂、哪里有重复代码
- 🔒 **安全漏洞报告**：有没有硬编码密码、危险的 eval 调用、有漏洞的依赖包
- 📦 **依赖风险报告**：哪些包该升级了
- 🌐 **API 接口报告**：接口有没有鉴权、有没有做输入校验

每个报告都有 1–10 分的评分，还有按优先级排列的修复建议，一目了然。

---

## 效果长这样

分析完成后 Claude 会先给你一个执行摘要：

```
## 🔍 项目分析完成

技术栈：Python 3.11 + FastAPI
架构类型：单体 REST API

📊 整体健康度：5.8/10

| 维度       | 评分   | 状态 |
|------------|--------|------|
| 代码质量   | 6/10   | 🟡   |
| 安全性     | 4/10   | 🟠   |
| 可测试性   | 3/10   | 🔴   |
| 依赖健康度 | 7/10   | 🟢   |

🚨 关键发现
1. 发现 3 处硬编码 API Key（security.md 第 2 节）
2. 14 个函数圈复杂度 > 10，建议重构
3. 测试覆盖率接近 0%，核心业务逻辑无测试保护
```

然后所有详细报告都作为文件输出，可以直接下载。

---

## 安装方式

在 Claude 的 skills 目录放一个文件夹就行了，三个文件，没有任何依赖：

```
repo-test-analyst/
├── SKILL.md
└── references/
    ├── testing-strategies.md
    └── report-templates.md
```

装好之后直接跟 Claude 说"帮我分析这个项目"就能触发，不用记任何命令。
